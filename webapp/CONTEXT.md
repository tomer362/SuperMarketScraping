# Web App Context

## Product And UX Notes

- Global search already lives in the authenticated top shell. Do not add empty-state cards that explain where the search is or how to use basic page controls.
- Prefer quiet, useful empty states only when they answer a real user question or unblock an action. Avoid filler copy such as “search from the top bar” when the UI already makes that obvious.
- The search/results page should stay focused on results, validation states, loading states, and meaningful no-results feedback.
